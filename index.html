### What is Reactant.jl? - Two array types: `ConcreteRArray` and `TracedRArray`. - `@compile` macro. ```julia using Reactant x_reactant = Reactant.to_rarray(x) # returns ConcreteRArray f_reactant = @compile f(x_reactant) f_reactant(x_reactant) ```
### How Reactant works? 1. Capture Julia array programs in MLIR. 1. Optimize, run AD, optimize again. 1. Evaluate on the XLA runtime (used by JAX/Tensorflow).
### What is Reactant.jl? - Capture high-level array operations to MLIR. - Optimizing compiler built on top of MLIR. - `>`200 Linear Algebra optimizations. - Device Retargeting. - Leverage the XLA runtime for evaluation. - Compatible with mutation, control-flow, autodiff (Enzyme). - Multi-device execution (Sharding & MPI). - JAX interoperability.

Device retargeting from CUDA


                    using Reactant, CUDA

                    function square_kernel!(x, y)
                        i = threadIdx().x
                        x[i] *= y[i]
                        return nothing
                    end

                    function square!(x, y)
                        @cuda blocks = 1 threads = length(x) square_kernel!(x, y)
                        return nothing
                    end

                    Reactant.set_default_backend("cpu")

                    oA = collect(1:1:64)
                    A = Reactant.to_rarray(oA)
                    B = Reactant.to_rarray(100 .* oA)
                    @jit square!(A, B)

                    Reactant.set_default_backend("tpu")

                    oA = collect(1:1:64)
                    A = Reactant.to_rarray(oA)
                    B = Reactant.to_rarray(100 .* oA)
                    @jit raise=true square!(A, B)
